GIVEN `tools/audit-shells.mjs` checa cada `service.slug` e `<handle>/index.html`
mas NÃO checa portfolio slugs, AND um poem ou essay adicionado em
`profile.yaml#portfolio[]` sem `<handle>/<slug>/index.html` correspondente
silenciosamente 404 quando o usuário clicar (L-007),

WHEN extendemos `audit-shells.mjs` pra também iterar
`people[].portfolio[]` (não-draft) e verificar que `<handle>/<slug>/index.html`
existe pra cada,

THEN audit captura gap em qualquer kind de portfolio (poem hoje, essay
após AL-27, outros futuros) — bake/PR não passa silenciosamente com
URLs órfãs.

## Implementação

Em `tools/audit-shells.mjs`, adicionar bloco após o check de handles:

```js
// 3. Cada portfolio item (não-draft) → <handle>/<slug>/index.html
for (const p of (AL.people || [])) {
    if (!Array.isArray(p.portfolio)) continue;
    for (const item of p.portfolio) {
        if (item.draft) continue;
        const shell = path.join(ROOT, p.handle, item.slug, "index.html");
        if (!fs.existsSync(shell)) {
            gaps.push({
                kind: `portfolio:${item.kind}`,
                handle: `${p.handle}/${item.slug}`,
                expected: shell,
                ref: item.titulo,
            });
        }
    }
}
```

Mensagem de erro inclui kind (poem|essay) + handle/slug pra desambiguar.

Template HTML pra criar shell (em mensagem de fix):
```html
<!DOCTYPE html><html lang="pt-BR"><head>
<meta charset="UTF-8"><meta name="viewport" content="width=device-width,initial-scale=1.0">
<title>{titulo} — {nome} — Arte Longa</title>
<script src="/assets/bootstrap.js"></script>
</head><body data-page="{kind}" data-slug="{slug}"></body></html>
```

## Critérios de aceitação

- [x] `tools/audit-shells.mjs` itera portfolio + flags gaps.
- [x] Mensagem de erro distingue kind (poem vs essay).
- [x] Drafts pulados (não disparam falha).
- [x] Run no estado atual passa (Inocência tem shell, Mães tem shell).
- [x] Manual test: inject portfolio entry sem shell → audit falha.

## Out of scope

- Auto-gerar shells faltantes (gerar shell HTML é um one-liner manual; automação é AL futura se valer a pena).
- Audit pra outros kinds não-existentes (song, video) — adicionar quando o kind for criado.
