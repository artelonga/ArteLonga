GIVEN site é estático leve mas não há mensuração formal, AND adicionamos
bundles (renderer.js bundle 77KB, css 3 files, data.js ~1500 lines) que
crescem sem alerta, AND regressão de performance só seria pega em
smoke test manual,

WHEN configuramos Lighthouse CI rodando contra páginas principais em
cada PR, AND definimos budget (LCP < 2s, CLS < 0.1, bundle size caps),
AND falha de budget bloqueia merge,

THEN performance degradation é caught em PR review, não em prod.

## Budget proposto

| Métrica | Budget |
|---|---|
| LCP | < 2.0s |
| CLS | < 0.1 |
| FID/INP | < 200ms |
| TTI | < 3.0s |
| Total JS bundle | < 150 KB gzipped |
| Total CSS | < 50 KB gzipped |
| Lighthouse Performance | ≥ 90 |
| Lighthouse SEO | ≥ 95 |
| Lighthouse Accessibility | ≥ 90 (linked AL-41) |

## Implementação

1. `.lighthouserc.js` (config Lighthouse CI).
2. GH Action `.github/workflows/lhci.yml` roda em PRs.
3. Páginas auditadas: `/`, `/parceiros/`, `/yuri/`, `/servicos/grafite/`, `/contato/`.
4. Budget violations comentadas no PR via bot.

## Critérios de aceitação

- [ ] Lighthouse CI configurado.
- [ ] GH Action roda em PRs.
- [ ] Baseline scores documentados.
- [ ] Budget enforcement (regression bloqueia merge).
- [ ] CLAUDE.md doc sobre performance budget.
