GIVEN AL-45 LHCI quality pipeline foi estabelecido com budgets
aspirational (perf ≥ 0.9, seo ≥ 0.95, CLS ≤ 0.1, axe color-contrast
enabled), MAS baseline real do site não atinge esses números, AND no
AL-48 PR #54 os budgets foram relaxados pro baseline atual pra
desbloquear o merge (mesmo padrão de "fix later" do color-contrast),

WHEN fechamos a lista de baseline issues uma a uma:
1. `color-contrast` axe rule (disabled em AL-45) — re-enable após bumpar
   greys `#888`/`#999` pra atender WCAG AA 4.5:1 contraste.
2. SEO 0.92 → 0.95 nas 3 páginas afetadas (/yuri/, /servicos/grafite/,
   /contato/) — provavelmente meta description faltando, alt text em
   images, ou heading hierarchy.
3. CLS 0.224 em /contato/ — bug real de layout shift no form. Investigar
   (form pills toggle? imagens sem width/height?).
4. Performance 0.89 em /contato/ — provavelmente combinação de bundle
   size + render-blocking + CLS. Endereçar 3 ajuda 4.

WHEN bumpamos LHCI budgets de volta pros valores aspirational:
- `categories:performance` ≥ 0.9
- `categories:seo` ≥ 0.95
- `cumulative-layout-shift` ≤ 0.1
- re-enable axe `color-contrast` rule em `tests/e2e/smoke.spec.ts`

THEN site atinge baseline de qualidade contemporâneo e LHCI gates
prevenem regressão futura.

## Sub-tasks (ordem sugerida)

### Phase 1: A11y contrast (baseline issue do AL-45)

- Bump greys no CSS: `#888` (~3.5:1) → `#666` (~5.7:1); `#999` (~3:1)
  → `#5a5a5a` (~5:1). Manter scale visual coerente.
- Re-enable `color-contrast` rule em `tests/e2e/smoke.spec.ts` (remover
  do disable list).
- Verify axe zero violations em todas as 9 pages.

### Phase 2: SEO close (0.92 → 0.95)

Investigar com Lighthouse local: `npx lhci collect --url http://localhost:8000/yuri/`. Provavelmente:
- Missing `<meta name="description">` em algumas pages (AL-47 cobriu? double-check).
- Images sem `alt` text.
- Heading order/skip (h1 → h3 sem h2).
- Tap targets pequenos (mobile).

Fix iterativo. Cada page atinge ≥ 0.95.

### Phase 3: CLS close (/contato/ 0.224 → ≤ 0.1)

Investigar `/contato/`: provavelmente layout shift do form pills toggle
(físico/digital) ou imagens sem `width`/`height`. Lighthouse trace
mostra elementos shifting.

Mitigações típicas:
- `width`/`height` attributes em `<img>`.
- CSS `aspect-ratio` pra placeholders.
- Skeleton / reserved space pra content que carrega async.
- Avoid late font-loading FOUT (já é system font, OK).

### Phase 4: Perf close (0.89 → 0.9 em /contato/)

Provavelmente resolvido pelas Phases 1-3 (CLS é parte do score). Se
ainda gap, investigar:
- Render-blocking CSS (criticar bootstrap.js chain).
- Unused JS no bundle.
- Image optimization (formats, sizes).

### Phase 5: Re-enable LHCI assertions como `error` (não `warn`)

Após AL-48, LHCI ficou como **warn-only** em `.lighthouserc.cjs` por
noise de shared CI runner (perf 0.89↔0.76 entre runs back-to-back).
AL-49 precondition pra re-enable como `error`:

- Baseline issues fechadas (Phases 1-4 acima) — perf score sai do limite
- Possibly migrate LHCI pra runner mais consistente:
  - GH Actions `larger-runners` (paid, more consistent CPU)
  - Self-hosted runner com infra dedicada
  - LHCI Server (own dashboard) com retry logic
- OU manter warn-only como decisão consciente, registrar em L-022+

Critério: 5 runs back-to-back do mesmo SHA produzem variance < 5%
nos scores. Aí pode confiar em `error`.
