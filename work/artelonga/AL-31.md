GIVEN site não tem testes automatizados, AND regressões só são pegas
em smoke test manual pós-merge, AND alguns bugs (L-001 DCL race, L-002
silent fail, L-007 404) afetaram páginas inteiras silenciosamente
antes de detecção,

WHEN adicionamos suite Playwright minimal cobrindo as 8 páginas-canário
do site, AND wire em GH Action que roda em PRs, AND testes simples
verificam: (a) status HTTP 200, (b) elementos-chave renderizam,
(c) console sem erros, (d) nenhum L-002 fallback ("Algo quebrou")
visível,

THEN PR que quebra alguma página principal não merge — falha aparece
no review com link pro Playwright trace.

## Páginas-canário

| Path | Verifica |
|---|---|
| `/` | hero "Serviços" + busca input + ≥1 chip sup-cat + ≥1 card |
| `/parceiros/` | roster com ≥10 pessoas listadas |
| `/yuri/` | h1 com "Yuri", bio paragraph, link pra poema Mães |
| `/yuri/maes/` | título "Mães", 4 estrofes |
| `/kiyoshi/inocencia/` | título "Inocência", stanzas |
| `/quilomboaraucaria/` | nome + servicos[] visíveis + membros[] |
| `/servicos/grafite/` | título "Grafite" + responsavel |
| `/contato/` | form com nome+email+mensagem inputs + privacy notice |
| `/design/` | swatches de cor + ≥1 component canvas visível |

## Implementação

```ts
// tests/e2e/smoke.spec.ts
import { test, expect } from '@playwright/test';

const BASE = process.env.BASE_URL || 'http://localhost:8000';

const pages = [
    { path: '/', expect: { h1: /Serviços|Marketplace/ } },
    { path: '/yuri/maes/', expect: { stanzaCount: 4 } },
    // ... etc
];

for (const p of pages) {
    test(`smoke ${p.path}`, async ({ page }) => {
        const messages: string[] = [];
        page.on('console', m => messages.push(m.text()));
        const resp = await page.goto(BASE + p.path);
        expect(resp?.status()).toBe(200);
        const fallback = await page.locator('text=Algo quebrou').count();
        expect(fallback).toBe(0);
        const errors = messages.filter(m => m.includes('error'));
        expect(errors).toHaveLength(0);
        // page-specific assertions...
    });
}
```

GH Action:
```yaml
# .github/workflows/smoke.yml
name: smoke
on: [pull_request, push]
jobs:
  smoke:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
      - run: npm ci
      - run: npm run build
      - run: npx playwright install --with-deps chromium
      - run: npx serve . -p 8000 &
      - run: BASE_URL=http://localhost:8000 npx playwright test
      - uses: actions/upload-artifact@v4
        if: failure()
        with: { name: playwright-trace, path: test-results/ }
```

## Critérios de aceitação

- [ ] `playwright.config.ts` + `tests/e2e/smoke.spec.ts` existem.
- [ ] 8+ páginas-canário cobertas.
- [ ] `npm run test:e2e` roda local contra `http://localhost:8000`.
- [ ] GH Action roda em PRs + falha bloqueia merge.
- [ ] Failure produz Playwright trace artifact.
- [ ] CLAUDE.md doc seção "Running tests".

## Out of scope

- Visual regression (screenshots diff). Pode ser extension futura.
- Performance tests (LCP/CLS) — Tier 5 separadas.
- Mobile viewport testing — começa desktop, adiciona mobile depois.
