GIVEN `data.js#solutions` (alias "universos") tem 5+ entries hardcoded
incluindo bundled services, platforms, lifecycle (active|futuro), AND
estrutura é rica o suficiente pra justificar YAML per item,

WHEN movemos cada solution pra `solucoes/<slug>/solution.yaml` + bake
`tools/bake-solutions.mjs`,

THEN solutions editáveis por folder. URL `/solucoes/<slug>/` continua
funcionando via shell HTML existente + renderer.

## Critérios de aceitação

- [ ] `solucoes/<slug>/solution.yaml` files criados.
- [ ] bake-solutions regenera AUTO-GENERATED:SOLUTIONS-START/END em data.js.
- [ ] validate-yaml extends --type=solution.
- [ ] Schema Solution em openapi.yaml.
- [ ] V bumped.
