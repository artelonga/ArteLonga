GIVEN cada um dos ~150 HTML shells (`<handle>/index.html`,
`servicos/<slug>/index.html`, etc) tinha próprios `<link rel="stylesheet">`
e `<script src="...?v=20260509">` referenciando 4-5 assets cada, AND
bumpar versão pra invalidar cache exigia editar 600+ entries (150
arquivos × 4-5 referências), AND um asset que mudava forçava ou
edição em massa ou cache stale por usuários,

WHEN extraímos toda a lógica de loading pra `assets/bootstrap.js`
(único `<script>` em cada shell), AND o bootstrap injeta CSS + scripts
dinamicamente com `?v=V` onde V é uma única constante no topo do file,

THEN bumpar V em UM lugar invalida cache de TODO o site. Adicionar
asset novo é uma linha em bootstrap.js, não 150 PRs em shells.

## Critérios de aceitação

- [x] Cada shell HTML tem só `<script src="/assets/bootstrap.js"></script>` (sem mais references diretas a assets versionados).
- [x] `bootstrap.js` define `var V = "..."` como single point of truth.
- [x] CSS files (`site.css`, `components.css`, `pages.css`) carregados via injection.
- [x] Scripts (`analytics.js`, `data.js`, `renderer.js`) carregados via injection com `async = false`.
- [x] Site funcional após o refactor (regressão check em main pages).
