GIVEN site é público e Yuri/parceiros são referenciados em buscas,
AND não há JSON-LD structured data, OpenGraph cards, ou sitemap.xml,
AND páginas em search results aparecem com snippet genérico,

WHEN adicionamos por página:
- JSON-LD Schema.org (Person, LocalBusiness, Service, CreativeWork pra poems)
- OG meta tags (title, description, image, url)
- Twitter cards
- `sitemap.xml` gerado pelo bake
- `robots.txt` declarando sitemap

THEN search engines indexam com rich snippets, share em social
mostra preview, crawlers descobrem URLs.

## Implementação

1. Renderer: cada page module emite `<head>` extra (OG + JSON-LD)
   apropriado ao content.
2. `tools/bake-sitemap.mjs` — gera `sitemap.xml` listando todas as URLs
   do site (handles, services, poems/essays). Roda no build.
3. `robots.txt` (já existe? checar) com `Sitemap: https://artelonga.com.br/sitemap.xml`.
4. OG image default (logo Arte Longa) + per-handle override (pic).

## Schemas-alvo

- Home: WebSite + Organization (Arte Longa)
- Profile: Person (com sameAs, jobTitle, image)
- Service: Service (provider, areaServed)
- Poem: CreativeWork (author, datePublished)
- Community: Organization

## Critérios de aceitação

- [ ] Renderer emite OG + JSON-LD apropriado por page type.
- [ ] `sitemap.xml` gerado + cobre todas URLs.
- [ ] `robots.txt` declara sitemap.
- [ ] Lighthouse SEO score ≥ 95 em páginas principais.
- [ ] Manual test: Twitter Card Validator + Facebook Sharing Debugger OK.

## Out of scope

- AMP.
- Pre-rendering pra SSR — page é estática + SPA-style hydration funciona.
- Hreflang (depende AL-40 i18n).
