GIVEN site é público e Yuri/parceiros/services são searched, AND não
há JSON-LD structured data, OG meta tags, ou sitemap.xml, AND páginas
em SERP aparecem com snippet genérico (preview sem image, sem schema
rich result), AND social shares (WhatsApp, LinkedIn) não tem preview
card,

WHEN cada page module em `src/pages/` emite `<head>` específico com:
- OG meta (og:title, og:description, og:image, og:url, og:type)
- Twitter cards (twitter:card, twitter:title, etc)
- JSON-LD Schema.org apropriado ao tipo de page
- Canonical URL
- AND adicionamos `tools/bake-sitemap.mjs` que gera `sitemap.xml`
  listando todas as URLs (home, parceiros, services, profiles, poems,
  essays, communities, missions, solutions), AND `robots.txt` declara
  o sitemap,

THEN search engines indexam com rich snippets (Person card pra perfil,
Service breadcrumb pra catálogo), social share mostra preview
profissional, crawlers descobrem todo o site sem depender de internal
links.

## JSON-LD schemas por page type

| Page | Schema.org type | Campos chave |
|---|---|---|
| Home (`/`) | WebSite + Organization | name, url, logo, description, sameAs |
| Profile (`/<handle>/`) | Person | name, image, jobTitle, sameAs (social), worksFor |
| Profile (business type) | LocalBusiness | name, address, telephone, image |
| Service (`/servicos/<slug>/`) | Service | name, description, provider (Organization), areaServed, serviceType |
| Community (`/<handle>/`) | Organization | name, url, logo, member, foundingDate |
| Poem (`/<handle>/<slug>/`) | CreativeWork (Poem) | name, author (Person), datePublished, text |
| Essay (`/<handle>/<slug>/`) | Article + author | headline, author, datePublished, articleBody |

## OG image strategy

- Default: `/assets/og-default.png` (logo Arte Longa em fundo branco).
- Per-handle: se `<handle>/profile.yaml.pic` existe, usa como OG image.
- Per-service: default + texto overlay? Out of scope inicial. Use default.

## Sitemap generator

`tools/bake-sitemap.mjs`:

```js
import fs from 'fs';
import { fileURLToPath } from 'url';

global.window = {};
require('./assets/data.js');
const AL = global.window.AL;

const urls = ['/'];
for (const p of AL.people) urls.push(`/${p.handle}/`);
for (const c of AL.communities) urls.push(`/${c.handle}/`);
for (const s of AL.services) urls.push(`/servicos/${s.slug}/`);
for (const m of AL.missions) urls.push(`/missoes/${m.handle}/`);
for (const sol of AL.solutions) urls.push(`/solucoes/${sol.handle}/`);
for (const p of AL.people) {
    for (const item of (p.portfolio || [])) {
        if (!item.draft) urls.push(`/${p.handle}/${item.slug}/`);
    }
}
// Sort + dedupe
const sorted = [...new Set(urls)].sort();

const xml = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
${sorted.map(u => `  <url><loc>https://artelonga.com.br${u}</loc></url>`).join('\n')}
</urlset>`;
fs.writeFileSync('sitemap.xml', xml);
```

`robots.txt` (criar se ausente):
```
User-agent: *
Allow: /
Sitemap: https://artelonga.com.br/sitemap.xml
```

## Implementation em renderer

Cada `src/pages/<page>.ts` retorna `(metaTags: string[])` extra além de
HTML body. Dispatcher injeta os meta tags no `<head>` antes do render.

Ou: cada page module mutates `document.head` no `render()` adicionando
`<meta>` + `<script type="application/ld+json">` antes de mutar body.

Decisão de design fica pra implementador.
