GIVEN `docs/STATE.md` é gerado por `tools/bake-state.mjs` rodando manual
(AL-3), AND fica stale conforme o CHANGELOG cresce, AND ninguém lembra
de rodar `node tools/bake-state.mjs` periodicamente,

WHEN adicionamos `.github/workflows/bake-state.yml` que roda no cron
mensal (`0 3 1 * *` — 3am dia 1) + manual via `workflow_dispatch`,
checa out, roda `npm ci && npm run bake-state` (ou node tools direct),
commita + push se diff,

THEN STATE.md fica atualizado sem intervenção. Onboarding sempre lê
snapshot recente.

## Implementação

```yaml
# .github/workflows/bake-state.yml
name: bake-state
on:
  schedule: [{ cron: '0 3 1 * *' }]
  workflow_dispatch: {}
permissions: { contents: write }
jobs:
  bake:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20' }
      - run: npm ci
      - run: node tools/bake-state.mjs
      - uses: stefanzweifel/git-auto-commit-action@v5
        with:
          commit_message: "chore(state): auto-bake STATE.md"
          file_pattern: docs/STATE.md
```

## Critérios de aceitação

- [ ] `.github/workflows/bake-state.yml` existe + valida sintaticamente.
- [ ] Manual trigger via `gh workflow run bake-state.yml` produz commit em main (se houve drift).
- [ ] Cron schedule documentado em `CLAUDE.md`.
- [ ] Sem commit redundante se STATE.md já em sync.

## Out of scope

- bake-state pra outros docs (LESSONS é append-only humano, não auto-gen).
- Notifications (Slack etc) quando bake roda.
- Generation on every push em CHANGELOG.md — mensal basta.
