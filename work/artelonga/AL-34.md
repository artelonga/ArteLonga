GIVEN home renderiza serviços por ordem de declaração em data.js
(não por popularidade), AND AL-33 vai produzir
`assets/popularity.json` com `{ path: views, visitors }` per service,

WHEN o `src/pages/home.ts` lê `popularity.json` no init, AND sort
`AL.publicServices()` (apenas top-level) por `views DESC` antes de
renderizar grid + chips,

THEN serviços mais procurados aparecem primeiro. Sinal empírico
substitui ordem arbitrária.

## Blocked by

**AL-33** — sem `popularity.json` no repo, não há dado pra rankear.

## Implementação (quando desbloquear)

```ts
// src/pages/home.ts
import popularity from '/assets/popularity.json' assert { type: 'json' };

const byPath: Map<string, number> = new Map(
    popularity.items.map(i => [i.path, i.views])
);
const topo = AL.publicServices()
    .filter(s => !s.parent)
    .sort((a, b) => {
        const va = byPath.get(`/servicos/${a.slug}/`) || 0;
        const vb = byPath.get(`/servicos/${b.slug}/`) || 0;
        if (va !== vb) return vb - va;
        return a.titulo.localeCompare(b.titulo, 'pt-BR');
    });
```

Fallback: se popularity.json missing ou empty, usa alfabético (current behavior).

## Critérios de aceitação

- [ ] home grid ordenado por popularity desc.
- [ ] Tie-break alfabético pt-BR.
- [ ] Fallback: missing/empty popularity.json → ordem default (não crash).
- [ ] Filter (search + supercat + alOnly) preserva ranking dentro do resultado.
