GIVEN AL-26 estabeleceu `PortfolioItem = oneOf [PortfolioPoem, PortfolioEssay]`
em `profile.yaml`, AND Alice tem 8 essays placeholder inline em
`alice/profile.yaml` num shape ad-hoc (`essaysTitle: ...` + `essays: [{ titulo, short, long }]`),
AND essa duplicação de shape (essays inline vs portfolio[]) impede a
abstração unificada,

WHEN movemos os 8 essays de Alice pro `portfolio[]` array com
`kind: essay`, AND removemos `essaysTitle` + `essays[]` legacy fields,
AND adicionamos `src/pages/essay.ts` espelhando `poem.ts` (renderer pra
`<handle>/<slug>/` com `body=essay` data-page),

THEN essays consumem a mesma infra de poems (bake derive + shell HTML
template + URL pattern). Renderer suporta os dois `kind` via dispatcher.

## Estado atual

`alice/profile.yaml`:
```yaml
essaysTitle: Ensaios · Pensamento Islâmico
essays:
  - titulo: ''
    short: ''
    long: ''
  - titulo: ''   # × 8 placeholders
```

`src/pages/poem.ts` existe; `essay.ts` não.

## Estado-alvo

```yaml
portfolio:
  - kind: essay
    slug: <slug>
    titulo: <titulo>
    short: <teaser>
    body: |
      <markdown multi-linha>
    draft: true   # enquanto placeholder
```

`src/pages/essay.ts`:
- Lê `AL.get(handle).portfolio` filtrado por `kind=essay && slug===target`.
- Renderiza titulo + body (mdInline ou markdown.js) + back link.
- Mesmo padrão visual de poem.ts (typography editorial).

`assets/data.js`: se bake-people criar bloco `AUTO-GENERATED:ESSAYS-START/END`
similar a POEMS — derivado de people[].portfolio[kind=essay]. Ou só expor
via `AL.essaysByAuthor(handle)` lendo direto do portfolio.

## Implementação

1. Schema `PortfolioEssay` já existe em openapi (AL-26).
2. Alice profile.yaml: substitui `essaysTitle` + `essays[]` por `portfolio: [{kind: essay, draft: true, slug, titulo, short, body}]` × 8.
3. `src/pages/essay.ts` novo.
4. `src/dispatcher.ts` adiciona case `essay: () => import('./pages/essay.ts')`.
5. `<handle>/<slug>/index.html` shells: criar pras essays não-draft (provavelmente nenhuma inicialmente — todas placeholders).
6. bake-people: opcional, derivar `essays[]` em data.js similar a poems[] (ou só ler direto do portfolio).
7. CHANGELOG + V bump + AL-27 status:done.

## Critérios de aceitação

- [ ] `alice/profile.yaml` sem `essaysTitle`/`essays[]`; tem `portfolio[]` com kind=essay × 8.
- [ ] `src/pages/essay.ts` renderiza essay individual em `/alice/<slug>/`.
- [ ] Renderer dispatcher trata `data-page="essay"`.
- [ ] `npm run audit` passa (incluindo audit-shells se for atualizado em AL-28).
- [ ] Validate-yaml passa (PortfolioEssay schema OK).
- [ ] V bumpado.

## Out of scope

- Essays publicados com texto real (placeholders ficam draft).
- Migrar outros profile.yamls que possam ter shape similar.
- Style/typography fine-tuning (segue padrão poem.ts).
