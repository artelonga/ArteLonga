GIVEN o site tinha 50+ páginas HTML estáticas duplicando header/footer/
estrutura inteira, AND mudar layout exigia editar dezenas de arquivos
manualmente (com risco de typos como "U<handle>" em 21 perfis), AND
adicionar membro novo era criar HTML completo do zero,

WHEN movemos toda lógica + dados pra dois arquivos centrais
(`assets/data.js` com pricing/people/services/communities/missions,
`assets/renderer.js` com dispatcher por `body[data-page]`), AND HTML
shells viraram entry points de 11 linhas com `data-page` + `data-handle`,

THEN adicionar/editar dado é UM lugar (data.js). Adicionar perfil é
copiar shell template + adicionar entry em data.js. Mudança de layout
é UM render function em renderer.js.

## Critérios de aceitação

- [x] `assets/data.js` exporta `window.AL` com all data + helpers (`AL.get`, `AL.publicServices`, etc).
- [x] `assets/renderer.js` tem `pageFns` dispatcher (home, parceiros, profile, service, etc).
- [x] HTML shells ≤ 15 linhas, só com `<body data-page="...">`.
- [x] Component CSS modulado (`site.css`, `components.css`, `pages.css`).
- [x] Site renderiza identicamente ao estado pré-refactor.
