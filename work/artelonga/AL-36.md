GIVEN ~50 services hardcoded em `data.js#serviceCatalog[]`, AND mesmo
princípio de AL-1/AL-2 (separação content/form via per-handle YAML)
não foi aplicado pra services ainda, AND adicionar/editar service
exige tocar arquivo gigante (vs editar só o YAML do service),

WHEN movemos cada service pra `servicos/<slug>/service.yaml`, AND
adicionamos `tools/bake-services.mjs` que regenera bloco
`AUTO-GENERATED:SERVICES-START/END` em data.js a partir dos YAMLs,
AND adicionamos schema `Service` validation no openapi.yaml (já
parcialmente lá — refinar),

THEN cada service tem source of truth próprio no folder dele, +
shell HTML pode ser auto-gerado pelo bake (mata L-007 de raiz).

## Estado atual

```
data.js:
  const serviceCatalog = [
    { titulo: "Desenvolvimento de API", parent: "...", implicitResponsavel: ["yuri"], ... },
    // × 50+
  ];
```

`servicos/<slug>/index.html` shells criados manualmente, vulneráveis a
gap (L-007).

## Estado-alvo

```
servicos/<slug>/
├── service.yaml   # NOVO — source of truth
└── index.html     # template padrão; opcionalmente auto-gerado pelo bake
```

```yaml
# servicos/desenvolvimento-de-api/service.yaml
titulo: Desenvolvimento de API
parent: Inteligência e Tecnologia
implicitResponsavel: [yuri]
paraQuem: "Empresas · times técnicos"
digital: true
hoursLow: 30
hoursHigh: 150
cnae:
  - { c: "6202-3/00", d: "Desenvolvimento e licenciamento de software customizável" }
```

bake-services lê + gera serviceCatalog em data.js + opcionalmente cria
shell HTML se não existe.

## Implementação

1. `tools/extract-services.mjs` (one-shot) lê data.js, gera ~50 service.yaml files.
2. `tools/bake-services.mjs` permanent — lê yamls, regenera bloco AUTO-GENERATED.
3. Schema Service em openapi.yaml já existe; refine se preciso (validate-yaml estende --type=service).
4. `tools/services-order.txt` pra ordem canônica.
5. `data.js#serviceCatalog` bloco AUTO-GENERATED.
6. Opcional: bake auto-cria `servicos/<slug>/index.html` shell se não existe (mata L-007).
7. Pre-commit + audit cobrem service.yaml drift.

## Critérios de aceitação

- [ ] ~50 `servicos/<slug>/service.yaml` files criados.
- [ ] bake-services determinístico.
- [ ] data.js mantém serviceCatalog em AUTO-GENERATED.
- [ ] validate-yaml extends --type=service.
- [ ] audit-shells captura missing shells (já feito; verificar).
- [ ] Site renderiza identicamente.
- [ ] V bumpado.

## Out of scope

- Migrar referências (citacoes.servico, etc) — já são strings de titulo.
- Renderer changes — consume mesma serviceCatalog shape.
