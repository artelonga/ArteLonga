GIVEN poemas autorais (Kiyoshi tem "Inocência") viviam num array
`poems = [...]` hardcoded em data.js, AND ensaios da Alice viviam
inline em alice/profile.yaml com fields ad-hoc (`essaysTitle`,
`essays[]`), AND queremos publicar mais peças (Yuri tem "Mães"), AND
não tinha abstração comum — cada tipo (poema, ensaio) tinha shape
diferente,

WHEN introduzimos `portfolio: PortfolioItem[]` em `Person` schema —
discriminated union por `kind` (poem|essay|...) — e migramos Kiyoshi
de `data.js#poems[]` pra `kiyoshi/profile.yaml.portfolio`, AND
adicionamos placeholder pra Yuri/"Mães" (draft: true até texto
chegar), AND `bake-people.mjs` deriva o array `poems[]` em data.js
(novo bloco AUTO-GENERATED:POEMS-START/END) lendo de
`people[].portfolio[kind=poem]`,

THEN content de cada autor vive em sua própria pasta (LGPD ownership
mantido, AL-1 principle escala pra portfolio), AND adicionar peça
nova é editar profile.yaml + bake (sem tocar data.js), AND a
abstração suporta novos tipos (song, video) sem mudança no shape.

## Schema (openapi/artelonga.yaml)

```yaml
PortfolioPoem:
  type: object
  properties:
    kind: { type: string, enum: [poem] }
    slug: { type: string }
    titulo: { type: string }
    stanzas: { type: array, items: { type: array, items: { type: string } } }
    draft: { type: boolean, default: false }
  required: [kind, slug, titulo, stanzas]

PortfolioEssay:
  type: object
  properties:
    kind: { type: string, enum: [essay] }
    slug: { type: string }
    titulo: { type: string }
    short: { type: string }
    body: { type: string }       # markdown
    draft: { type: boolean, default: false }
  required: [kind, slug, titulo]

PortfolioItem:
  oneOf: [{ $ref: PortfolioPoem }, { $ref: PortfolioEssay }]
  discriminator: { propertyName: kind }

Person:
  properties:
    # ... campos existentes ...
    portfolio:
      type: array
      items: { $ref: PortfolioItem }
```
