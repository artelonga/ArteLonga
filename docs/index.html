co serve

Sobe o servidor web do co localmente (localhost-first).

Inicia o servidor HTTP do co na sua máquina. Flags: --port (54321), --data-dir, --open, --public (bind 0.0.0.0).

onde se aplicaRodar o co na sua máquina; inspeção local antes de publicar.

co-auto

Motor de automação de tarefas (binário separado, dev/co-auto).

Lê o board work/<space>/<KEY>-<id>.md e orquestra agentes LLM no loop planejar→executar→revisar (roteamento de modelo por tarefa).

onde se aplicaEntrega contínua dirigida por tarefas.

co init <name>

Inicializa um novo espaço de conteúdo.

Cria a estrutura de um espaço novo com a key derivada do nome.

onde se aplicaComeçar um projeto.

co push · co sync push

Publica o conteúdo no servidor remoto (idempotente).

POST /api/v1/universes + PUT .../vault/{path}; delta jj por padrão. Flags: --dry-run, --bootstrap.

onde se aplicaPublicar/atualizar conteúdo.

co construir

Gera o site estático (Quartz) em public/.

Constrói o site público estático a partir do markdown (exclui _source/ com dados sensíveis).

onde se aplicaSite público estático.

co board

Sobe o quadro de gestão (Kanban/Calendar), porta 3000.

Abre o quadro de tarefas (visões Kanban e Calendar).

onde se aplicaQuadro de tarefas.

co launch

Transforma o diretório atual em espaço no co local.

Deriva a key do nome do diretório e sobe o conteúdo no co local na hora.

onde se aplicaSubir um MVP na hora.

co deploy

Publica o build num alvo (ex.: static-on-r2).

Faz o deploy do site estático no alvo configurado. rollback --deploy-id reverte.

onde se aplicaDeploy de site estático.

co auth token create

Cria token de API de 90 dias.

POST /api/v1/auth/token — emite um token portador para acesso programático.

onde se aplicaAcesso programático / API.

Lista completa em co --help e co help <tópico>.

③ API

Auth

JWT
POST /api/v1/auth/login Inicia sessão (magic link / credenciais).
POST /api/v1/auth/verify Verifica o código e emite o JWT.
POST /api/v1/auth/token Cria token de API de 90 dias.
GET /api/v1/auth/me Identidade da sessão atual.

Universes

JWT
GET /api/v1/universes Lista os espaços do usuário.
GET /api/v1/universes/public Lista os espaços públicos.
POST /api/v1/universes Cria um espaço (idempotente).
GET /api/v1/universes/{slug}/manifest Manifesto/metadados do espaço.

Conteúdo (entries)

JWT
GET·POST /api/v1/universes/{slug}/entries Lista ou cria entradas (markdown + frontmatter).
GET·PUT·DELETE .../entries/{*path} Lê, grava ou remove uma entrada.
GET .../changelog Histórico de mudanças do espaço.

Vault (compatível Obsidian)

token de API
GET·PUT·DELETE /api/v1/universes/{slug}/vault/{*path} Lê, grava ou remove um arquivo do vault.
POST .../vault/search Busca no vault (60 req/min).

Tempo real / sync

token de API
GET /api/v1/sync/ws WebSocket de sync CRDT.
GET /ws/doc/{slug}/{doc_id} WebSocket por documento.
GET /api/v1/events Stream de eventos (SSE).

Analytics

público
GET /api/v1/analytics/public/... Métricas públicas agregadas.
POST /api/v1/telemetry Ingestão de telemetria.

Feedback / contato

público
POST /api/v1/feedback Envia feedback.
POST /api/v1/contact Envia mensagem de contato.
